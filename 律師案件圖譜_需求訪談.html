一、專案基本資訊

專案名稱
律師案件圖譜
專案目標
打造可線上營運並具創業潛力的法律科技產品。

專案定位

依本次訪談結果，平台以公開判決、公開資訊、新聞聲量、社群討論與會員心得為基礎，協助有實際訴訟需求且具付費意願的使用者， 進行律師與事務所之初步篩選與比較的決策平台。

二、文件說明

本文件為現階段需求訪談結果之正式整理版本，用於彙整產品背景、目標使用者、核心需求方向、MVP 範圍、風險與待確認事項， 作為後續產品規劃、設計與實作的基礎參考。 訪談整理顯示，本文件非最終產品規格書，後續仍可能依資料可得性、法遵評估、商業策略與使用者回饋持續調整。

三、背景與問題定義

依訪談與討論整理，目前一般民眾在尋找律師時，常見痛點如下：

  1. 難以快速判斷某位律師是否真的有處理相似案件經驗
  2. 缺乏集中式平台整合律師案件輪廓、公開資訊與外部聲量
  3. 即使有公開判決資料，也不容易整理成可用的決策資訊
  4. 律師是否有爭議事件、負面新聞、社群討論、事務所風評等資訊分散且難比對
  5. 現有公開查詢工具偏官方、偏靜態，缺乏真正幫助決策的產品體驗
訪談結論：本專案希望建立一個能協助使用者「找律師」的資訊整合與付費決策平台，以降低蒐集成本並提升決策效率。

四、目標使用者

主要目標族群

有官司需求，且具一定經濟能力，願意付費換取資訊優勢的人

使用者特徵

  • 已經有法律問題或訴訟需求
  • 願意主動研究律師背景
  • 對案件適配度、誠信風險、口碑資訊有高度敏感度
  • 願意為更完整資訊、他人心得與整理內容付費

非主要族群

  • 僅作學術研究者
  • 單純法律新聞讀者
  • 高度依賴法扶體系者

五、產品核心價值主張

依本次訪談結果，本產品的核心價值不是單純列出律師名單，而是提供：

  1. 案件類型適配資訊：幫助使用者找出較符合自身案件類型的律師
  2. 公開風險與誠信觀測資訊：協助使用者初步辨識是否有公開爭議、負面新聞、負面社群聲量等風險訊號
  3. 事務所背景輔助判斷：若律師隸屬事務所，補充該事務所的公開風評與背景資訊
  4. 會員付費內容與心得交換機制：提供資訊解鎖模式，讓使用者可透過付費獲取更完整內容與會員心得

六、第一版產品目標（MVP）

MVP 目標

現階段規劃為建立可用的搜尋型平台，讓使用者透過複合條件查詢形成較符合自身需求的律師候選名單，並可看到：

  • 基本公開資訊
  • 案件類型偏好
  • 案件經驗輪廓
  • 公開爭議 / 風險資訊
  • 所屬事務所資訊
  • 會員心得與付費解鎖內容入口

MVP 案件範圍

  • 民事案件
  • 近五年資料
  • 刑事、行政案件與更長時間範圍留待後續階段評估

七、產品使用情境

情境 1：案件導向找律師

使用者已有特定法律問題，希望輸入案件類型與其他條件，篩選出較可能適合的律師。

情境 2：律師背景查核

使用者已經接觸某位律師，希望查看其公開案件輪廓、爭議資訊、新聞聲量與事務所背景。

情境 3：事務所與律師比對

使用者在多位律師間猶豫，希望透過案件經驗、公開風評與會員心得做進一步比較。

情境 4：付費解鎖決策資訊

使用者先看到免費摘要資訊，若有興趣，進一步付費解鎖會員心得、完整資訊與更細部整理內容。

八、核心功能需求

依訪談結果，以下內容為目前歸納出的初步需求方向，作為後續產品規劃與驗證的依據。

8.1 搜尋與查詢

依訪談結果，平台預期需支援 複合查詢（而非僅單一搜尋框），以符合使用者「多條件篩選」的使用需求。

初步整理如下：
  • 律師姓名
  • 案件類型
  • 地區
  • 是否有事務所
  • 事務所名稱
  • 公開爭議資訊
  • 新聞聲量
  • 社群負面討論
  • 案件經驗豐富度
現階段整理目標：協助使用者快速形成「較符合自身需求的律師候選名單」。

8.2 律師資料頁

現階段整理建議包含：每位律師可有專屬頁面，並區分免費可見與付費解鎖資訊，初步內容如下：

免費可見資訊 付費解鎖資訊
  • 律師姓名
  • 基本公開背景
  • 常見案件類型
  • 近五年民事案件輪廓
  • 所屬事務所
  • 基礎公開風險提示
  • 簡要資訊摘要
  • 更完整案件資訊
  • 更完整風險 / 爭議整理
  • 會員心得內容
  • 更細緻的比對與整理資料

8.3 風險 / 爭議資訊模組

訪談整理顯示，預期納入的來源可包含：新聞、社群、公開法律事件、事務所風評、其他公開可得資訊。

目的：作為使用者選擇律師時的 誠信風險輔助參考（以中性呈現為原則）。
建議正式名稱（避免「案底」等字眼）：
  • 公開風險資訊
  • 誠信風險觀測
  • 爭議事件整理
  • 公開聲量與風險觀測

8.4 事務所資訊模組

  • 事務所名稱
  • 事務所公開背景
  • 事務所相關風評
  • 事務所是否有爭議事件或負面聲量

8.5 會員與付費機制

訪談整理顯示，平台預期具備會員系統；商業模式方向參考 面試趣 類型（免費摘要 + 付費解鎖深入內容）。

付費解鎖內容傾向：
  • 會員心得
  • 完整資訊
  • 更花時間整理的獨門內容
  • 較難自行蒐集的資訊整合結果

金流：預計串接綠界。

8.6 會員心得機制

訪談討論方向：可採類似面試趣式的資訊交換 / 付費內容模式。

會員心得的角色：
  • 提升內容差異化
  • 形成平台黏著度
  • 作為付費價值來源之一
可能內容形式：
  • 與律師接觸經驗
  • 諮詢體驗
  • 委任流程經驗
  • 主觀感受與提醒事項

九、第一版不優先處理的項目

  1. LLM 深度分析：雖然未來有機會導入，但 MVP 暫不列為核心。
  2. 多模態分析：第一階段不考慮。
  3. 刑事與行政案件：先專注民事近五年。
  4. 過度主觀的律師能力評分：如戰力分數、勝率榜、最強律師排行等，不適合在 MVP 直接上線。

十、推薦邏輯初步方向

訪談中對「較合適律師」的判斷依據，初步整理為三類：

  1. 誠信與風險面：是否有公開法律事件、是否有負面社群聲量、是否有負面新聞話題
  2. 案件適配面：是否有相關案件類型經驗、是否處理過相似案件、該類案件的經驗是否相對豐富
  3. 組織背景面：是否隸屬事務所、事務所風評如何、事務所是否有可供參考的公開資訊

十一、資料來源方向

主要資料方向

  • 法院判決資料
  • 律師公開資訊
  • 新聞資料
  • 社群資料
  • 會員產出的內容
MVP 主要骨幹仍是：法院判決資料；其他資料來源用於增加決策層次與付費價值。

資訊分層建議

分層 內容
第一層：高可信公開資料 法院判決、官方律師公開資訊、主管機關或正式公告
第二層：中可信外部資料 主流新聞、可驗證媒體報導
第三層：低可信但有價值的輔助資料 社群討論、匿名會員心得、使用者經驗回饋
此分層用於後續 UI 呈現與權重設計討論，避免不同可信度資訊被等同看待。

十二、風險與法遵重點

訪談整理顯示，本專案需提早處理的重點（非僅技術）如下：

  1. 名譽與誹謗風險：需處理資料來源可靠性、資訊時間性、呈現方式中性化、當事人申訴與更正流程。
  2. 社群資料可信度風險：負面社群雜訊較高，需分級（主流媒體、官方公開資料、已判決公開資料、社群討論、匿名會員心得），不可視為同等可信度資訊。
  3. 會員心得風險：需設計審核規則、匿名規則、檢舉機制、申訴流程、下架機制與使用條款等配套。
  4. 推薦結果可解釋性：若平台呈現某律師「較適合」的判斷，需能說明依據，避免黑盒式評分造成風險與爭議。

十三、產品限制與聲明

依本次訪談結果，平台定位為資訊整理與決策輔助工具，相關使用邊界初步整理如下：

  • 本平台提供的是公開資訊整理、使用者經驗分享與輔助決策之參考內容。
  • 本平台不構成法律意見，亦不取代正式法律諮詢。
  • 本平台不構成訴訟結果保證。
  • 本平台不對律師專業能力作最終判定。
  • 使用者仍應自行評估並進行正式諮詢。

十四、MVP 產品描述

依本次訪談結果，律師案件圖譜定位為面向有實際訴訟需求使用者的法律科技平台，主要整合法院公開判決、律師與事務所公開資訊，以及新聞聲量與會員心得等資料來源。

現階段規劃中，使用者可先以複合條件查詢形成律師候選名單並進行初步比較，後續再透過付費機制解鎖更完整的決策參考資訊。

十五、待確認事項

以下項目尚未於本次訪談中完全定案，需於後續產品規劃與驗證階段進一步確認：

  1. 免費 / 付費資訊邊界
  2. 風險資訊分級與呈現規則
  3. 會員心得審核、檢舉、申訴與下架機制
  4. 推薦依據與可解釋欄位
  5. MVP 頁面骨架與資訊架構
  6. 上線初期是否開放會員心得投稿
